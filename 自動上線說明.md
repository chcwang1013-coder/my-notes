# 自動上線設定說明

設定一次，之後「檔案一改，網站自動更新」——不管是你自己改，還是請 Cowork 幫你改，都會自動同步。

## 運作原理

你的 Mac 背景會跑一個小程式，每分鐘檢查 `上線網站` 這個資料夾有沒有變動；只要有變動，就自動上傳到 GitHub，GitHub Pages 隨後自動重建網站。整個過程你不用做任何事。

> 重點：之後「網站的內容」都以 `上線網站/` 資料夾為準。要我改網站時，我就改這裡的 `photonics.html`、`blackhole.html`、`index.html`。

---

## 一次性設定（約 5 分鐘）

### 步驟 1：安裝並登入 GitHub Desktop
- 到 https://desktop.github.com 下載安裝，用你的 GitHub 帳號登入。
- （這一步是為了讓電腦拿到「推送權限」，之後自動上傳才不會被擋。沒有 GitHub 帳號就先在 github.com 免費註冊。）

### 步驟 2：在 GitHub 建立一個 repo
- 到 github.com → 右上 **＋ → New repository**。
- 名稱隨意（例如 `my-notes`），選 **Public**，按 Create。
- 建好後，複製它的網址（形如 `https://github.com/你的帳號/my-notes.git`）。

### 步驟 3：執行一鍵設定
- 打開 `上線網站` 資料夾，**雙擊 `一鍵設定.command`**。
  - 若跳出「無法開啟，因為來自未識別的開發者」：在該檔上按右鍵 →「打開」→「打開」。
- 依提示**貼上剛剛的 repo 網址**，按 Enter。
- 看到「推送成功」與「完成！」就對了。

### 步驟 4：開啟 GitHub Pages
- 到你的 repo → **Settings → 左側 Pages**。
- Source 選 **Deploy from a branch**，Branch 選 **main**、資料夾 **/(root)**，Save。
- 約 1 分鐘後，網址會出現在同一頁：`https://你的帳號.github.io/my-notes/`

完成！以後任何修改都會自動上線，約 1–2 分鐘生效。

## 常見問題

- **我改完多久會生效？** 約 1–2 分鐘（背景每分鐘檢查一次 + GitHub 重建）。
- **電腦關機會怎樣？** 關機期間不會上傳；開機後只要有未上傳的變動，會自動補上。
- **想暫停自動上傳？** 終端機執行：`launchctl unload ~/Library/LaunchAgents/com.owen.sitesync.plist`；要恢復把 `unload` 換成 `load`。
- **上傳有沒有成功的紀錄？** 看 `上線網站/.sync.log`。
- **網址是公開的嗎？** 是，任何人有網址都看得到。
